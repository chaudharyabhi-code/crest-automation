# Project Understanding Summary

## Project Overview
This is a **Playwright-based test automation framework** for a fintech application (Crest) that validates API responses against SQL database values with a specific comparison threshold.

## Key Architecture Components

### 1. Authentication System ([auth/](auth/))
- Handles user signup and OTP verification
- Manages access tokens for API authentication
- Global setup runs once before all tests

### 2. Test Structure ([tests/](tests/))
- Tests organized by feature (Dashboard, Portfolio, etc.)
- Each test calls an API, extracts values, and compares with SQL results
- Uses custom fixtures for authenticated API client

### 3. Comparison Logic ([utils/comparison.js](utils/comparison.js))
- Handles financial values with units (Cr, L, K)
- Converts between API format (e.g., "₹1.21 L") and raw DB numbers
- Implements threshold-based comparison (default 0.25%)

### 4. Database Integration ([utils/db/](utils/db/))
- PostgreSQL client for executing SQL queries
- SQL files stored in queries/ folder
- Dynamic user ID replacement in queries

### 5. Helper Functions ([utils/testHelpers.js](utils/testHelpers.js))
- `compareApiWithSql()` - Main comparison function
- `extractApiValue()` - Extracts values from nested API responses
- Generates formatted reports with pass/fail results

## Current Test Flow

1. **Authentication** happens once globally
2. **Test** calls API endpoint with authenticated client
3. **Extract** value from API response (handles nested JSON)
4. **Execute** SQL query with user ID
5. **Compare** values with 0.25% threshold
6. **Assert** test passes if difference < threshold

## Key Features

- **Unit-aware comparison**: Handles Cr (Crores), L (Lakhs), K (Thousands)
- **Threshold validation**: Configurable percentage difference allowed
- **Detailed reporting**: Shows API value, DB value, difference %, and pass/fail
- **Modular design**: Easy to add new endpoints and tests

## Comparison Logic Details

### Threshold Validation
- Default threshold: **0.25%**
- Formula: `|difference| / expected_value * 100`
- Test **PASSES** if difference < 0.25%
- Test **FAILS** if difference ≥ 0.25%

### Unit Conversion
- **Cr (Crores)**: Divides by 10,000,000
- **L (Lakhs)**: Divides by 100,000
- **K (Thousands)**: Divides by 1,000
- Automatically detects unit from API response
- Converts DB raw values to matching unit for comparison

## Adding New Test Cases

### Step 1: Add API Endpoint
Add to `endpoints/dashboard.js` or create new endpoint file:
```javascript
portfolioDetails: (userId) => `/api/v1/dashboard/portfolio-details?member_user_id=${userId}`
```

### Step 2: Create SQL Query
Add SQL file to `queries/` folder with `{USER_ID}` placeholder:
```sql
SELECT column_name FROM table WHERE user_id = {USER_ID}
```

### Step 3: Write Test Spec
Create test file in appropriate folder under `tests/`:
```javascript
test('Test Name', async ({ apiClient }) => {
  // 1. Call API
  const response = await apiClient.get(endpoint);

  // 2. Extract value
  const apiValue = extractApiValue(response.body, 'path.to.value');

  // 3. Compare with SQL
  const result = await compareApiWithSql({
    apiValue: apiValue,
    sqlFilePath: 'query.sql',
    userId: userId,
    sqlColumn: 'column_name'
  });

  // 4. Assert
  expect(result.comparison.pass).toBe(true);
});
```
